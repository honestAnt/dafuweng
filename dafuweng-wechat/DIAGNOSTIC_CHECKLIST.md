# 快速诊断检查清单

## ✅ 请按顺序检查以下项目

### 1. 项目类型检查

- [ ] 微信开发者工具左上角显示"小游戏"或"Game"
- [ ] 不是"小程序"

**如果不是小游戏项目：**
1. 关闭当前项目
2. 点击 "+" 导入项目
3. 选择"小游戏项目"
4. 重新导入

### 2. 文件结构检查

在项目目录中应该只有这些文件：

- [ ] `game.js` 存在
- [ ] `game.json` 存在
- [ ] `project.config.json` 存在

不应该有这些文件：

- [ ] `app.js` 不存在
- [ ] `app.json` 不存在
- [ ] `game.wxml` 不存在
- [ ] `game.wxss` 不存在
- [ ] `pages/` 目录不存在

### 3. 控制台检查

打开微信开发者工具的控制台：

- [ ] 点击屏幕时有输出：`Touch position: xxx xxx Screen: start`
- [ ] 没有红色错误信息

**如果没有输出：**
- 项目类型可能错误
- Canvas 可能未初始化

### 4. 屏幕显示检查

小游戏应该显示：

- [ ] 粉红色背景
- [ ] 标题："🍭 糖果大富翁 🍬"
- [ ] 玩家数量按钮（2、3、4、5）
- [ ] 20个角色卡片
- [ ] "开始游戏"按钮

**如果屏幕空白：**
- Canvas 可能未正确渲染
- 检查控制台错误

### 5. 按钮点击测试

点击玩家数量按钮：

- [ ] 控制台输出：`Player count X selected`
- [ ] 按钮颜色变化（选中变为红色）

点击角色卡片：

- [ ] 控制台输出：`Character X toggled`
- [ ] 卡片颜色变化（选中变为红色）

点击"开始游戏"按钮：

- [ ] 控制台输出：`Start game button clicked!`
- [ ] 切换到游戏界面

### 6. Canvas 尺寸检查

在控制台输入：

```javascript
console.log(canvas.width, canvas.height)
```

应该输出屏幕尺寸，例如：`375 667`

### 7. 游戏状态检查

在控制台输入：

```javascript
console.log(gameState)
```

应该输出游戏状态对象。

## 🚨 最常见的问题

### 问题：点击屏幕没有任何反应

**原因：** 使用了小程序项目而不是小游戏项目

**解决：**
1. 关闭当前项目
2. 重新导入，选择"小游戏项目"

### 问题：屏幕是空白的

**原因：** Canvas 未正确初始化

**解决：**
1. 清除缓存
2. 重新编译
3. 检查控制台错误

### 问题：按钮可见但点击无反应

**原因：** 触摸事件未正确注册

**解决：**
1. 检查控制台是否有触摸坐标输出
2. 如果没有，说明触摸事件未触发
3. 确认使用的是小游戏项目

## 📱 真机测试

如果开发者工具有问题，尝试真机测试：

1. 点击"预览"按钮
2. 用微信扫描二维码
3. 在手机上测试

真机测试可以排除开发者工具的问题。

## 🆘 如果所有检查都通过了但还是有问题

请提供以下信息：

1. 微信开发者工具的版本
2. 基础库版本
3. 控制台的完整输出
4. 点击屏幕时的触摸坐标
5. Canvas 的尺寸（在控制台输入 `console.log(canvas.width, canvas.height)`）

---

**记住：最重要的一点是确保使用"小游戏项目"！**
